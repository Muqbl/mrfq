> **Physical device QA is pending until tested on real devices and browsers. Browser viewport simulation is not production certification.**

Evidence rules:

- Mark Pass/Fail only after executing the workflow on the named physical device/OS and browser.
- Capture a screenshot for failures and for each role/dashboard baseline.
- Record OS/browser version, orientation, input method, console/network errors, and test data notes.
- Verify login, navigation, safe-area behavior, touch targets, scrolling, tables/forms, modals, camera/file input where supported, and logout.
