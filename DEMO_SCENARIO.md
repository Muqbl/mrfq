# سيناريو العرض التجريبي — Demo Scenario
**REGA Facility Care Prototype | نسخة تجريبية**

---

## تشغيل النظام قبل العرض

```bash
# 1. تثبيت المتطلبات
npm install

# 2. إنشاء بيانات تجريبية (الكلمات تُولَّد عشوائياً وتُطبع مرة واحدة)
npm run seed

# 3. تشغيل الخادم
npm start
```

قاعدة البيانات تُنشأ تلقائياً عند أول تشغيل. لا يوجد `data.db` في المستودع.

## الحسابات التجريبية / Demo Accounts

> الكلمات تُولَّد عند تشغيل `npm run seed` وتُطبع في الطرفية مرة واحدة فقط.
> يمكن تحديدها مسبقاً عبر متغيرات البيئة في `.env` (راجع `.env.example`).

| الدور | اسم المستخدم | الصلاحية |
|-------|-------------|---------|
| مدير النظام | `admin` | system_admin |
| مدير المرافق | `fm` | facility_manager |
| مدير النظافة | `manager` | cleaning_manager |
| مشرف النظافة | `supervisor1` | cleaning_supervisor |
| عامل نظافة | `worker3` | cleaner |

> ⚠ جميع الحسابات ستُطلب تغيير كلمة المرور عند أول تسجيل دخول.

## خطوات سيناريو العرض / Presentation Steps

### الخطوة 1 — تسجيل الدخول (مدير النظافة)
1. افتح `http://localhost:3000`
2. أدخل: `manager` + الكلمة من مخرجات `seed`
3. غيّر كلمة المرور عند المطالبة
4. ستظهر لوحة التحكم مع بانر **"نسخة تجريبية — بيانات غير حقيقية"**

### الخطوة 2 — إنشاء بلاغ نظافة
1. انتقل إلى **البلاغات** → **إنشاء بلاغ**
2. الموقع: "الردهة الرئيسية - الدور الأرضي"
3. العامل: "عامل 3" | الأولوية: عالية
4. اضغط **إنشاء بلاغ**

### الخطوة 3 — العامل يستلم المهمة
1. نافذة متصفح منفصلة (Incognito)
2. سجل دخول بـ `worker3` + كلمة المرور من `seed`
3. تظهر واجهة العامل المبسطة مع البلاغ المسند

### الخطوة 4 — العامل يرفع صورة
1. اختر الموقع من القائمة أو أدخل: `lobby-gf`
2. نفّذ البنود → **التقاط صورة** → **إرسال التقرير**
3. الصور تُحفظ كملفات على الخادم (لا base64 في DB)

### الخطوة 5 — المشرف يعتمد التقرير
1. عد لنافذة مدير النظافة → **التقارير**
2. ابحث عن التقرير (حالة: بانتظار الاعتماد)
3. اضغط **اعتماد** ✓

### الخطوة 6 — عرض التقرير المختصر
1. **لوحة التحكم** → إحصائيات اليوم، نسبة التغطية، مؤشر SLA
2. **تصدير CSV** أو **تصدير PDF**

## نقاط الأمان للتوضيح في العرض

| النقطة | التفاصيل |
|--------|---------|
| 🔐 **المصادقة** | HttpOnly SameSite=Strict Cookie — لا token في JS |
| 🗄 **قاعدة البيانات** | SQLite — sessions + audit_logs محفوظة في DB |
| 🛡 **Security Headers** | CSP, X-Frame-Options DENY, nosniff, Referrer-Policy |
| 📋 **Audit Log** | كل عملية مسجلة في جدول `audit_logs` داخل DB (لا ملف خارجي) |
| 👥 **الصلاحيات** | العامل يرى فقط مهامه — محمي من backend |
| ⏱ **Session Timeout** | انتهاء الجلسة بعد ساعتين — محفوظة في DB |
| 🔒 **كلمات المرور** | scrypt hash — لا نصوص واضحة في الكود |
| 🚫 **Rate Limiting** | 5 محاولات فاشلة/دقيقة فقط |
| 📁 **رفع الصور** | فحص Magic Bytes + UUID أسماء + حد 5MB |

## ملاحظات للعرض

- البانر البرتقالي "نسخة تجريبية" يظهر في جميع الصفحات
- لا توجد بيانات تشغيلية أو أسماء حقيقية في النظام
- استخدام الهوية البصرية لأغراض المحاكاة فقط
- المستودع لا يحتوي: `data.db` | `uploads/` | `node_modules` | `.env`
